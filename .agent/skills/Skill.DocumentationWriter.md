# Skill.DocumentationWriter

## Objective
Standardize all project documentation.

## Instructions
1. **Format**: Markdown (.md).
2. **Structure**:
   - **Title**: Clear and Descriptive.
   - **Overview**: What is this doc about?
   - **Sections**: Logical breakdown.
   - **Examples**: Code blocks or screenshots.
3. **Updates**:
   - Always update related docs when changing code (e.g., update API docs when modifying an endpoint).
